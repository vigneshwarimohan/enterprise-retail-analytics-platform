# Day 13 — Metadata Documentation

## Objective

The objective of Day 13 is to document table-level and warehouse-level metadata for the Enterprise Retail Analytics Platform.

## Metadata Areas Covered

The documentation covers:

- dataset names
- table names
- warehouse layers
- table purpose
- table grain
- source-to-table mapping
- refresh/load notes
- downstream usage
- row counts
- column counts

## Difference Between Data Dictionary and Metadata

The data dictionary explains the meaning of columns.

Metadata documentation explains the structure and management of tables, datasets, sources, layers, and usage.

## Tables Included

The metadata documentation includes:

- raw.sales
- raw.promotions
- raw.products_api
- staging.stg_sales
- staging.stg_promotions
- staging.stg_products

## Why This Matters

Metadata documentation helps users understand how the data warehouse is organised.

It improves maintainability, makes onboarding easier, and supports better governance.

## Day 13 Learning Summary

In Day 13, I documented the metadata for the Enterprise Retail Analytics Platform, including warehouse layers, table purpose, table grain, source mappings, row counts, column counts, and downstream usage.
