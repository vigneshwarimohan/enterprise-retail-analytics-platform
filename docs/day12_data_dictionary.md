# Day 12 — Data Dictionary

## Objective

The objective of Day 12 is to document the key tables and columns used in the Enterprise Retail Analytics Platform.

A data dictionary helps users understand what each table and column means.

## Tables Documented

The following tables were documented:

- raw.sales
- raw.products_api
- raw.promotions
- staging.stg_sales
- staging.stg_products
- staging.stg_promotions

## Documentation Areas

The data dictionary includes:

- table name
- layer
- grain
- source table
- key columns
- measures
- dimensions
- business meaning
- example values

## Why This Matters

A data dictionary makes the project easier to understand, maintain, and explain.

It helps developers, analysts, and business users understand how data should be interpreted.

## Day 12 Learning Summary

In Day 12, I created a data dictionary for the Enterprise Retail Analytics Platform. I documented table grain, important keys, measures, dimensions, and business meanings for the raw and staging layers.
